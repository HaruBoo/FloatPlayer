# FloatPlayer

🇯🇵 日本語 | [🇺🇸 English](./README.en.md)

macOS用の、常に最前面に浮くフローティングプレイヤーです。YouTube・写真(スクリーンショット等)・保存済み動画を、他のアプリで作業しながら同時に見続けられます。

Xcode不要、SwiftのコマンドラインツールだけでビルドできるネイティブのSwift/AppKit/SwiftUIアプリです。

---

## 特徴

- **YouTube再生** — 公式のIFrame Player APIを使った埋め込み再生(ダウンロードは行いません)。再読み込みなしで一時停止・再開・シークができます
- **写真/スクリーンショット表示** — ファイル選択・ドラッグ&ドロップ・クリップボード貼り付けに対応
- **動画ファイルのループ再生** — 保存済みの動画をBGV(バックグラウンドビデオ)のように流せます
- **スクリーンショットの自動フローティング** — Cmd+Shift+4などでスクショを撮った直後、UIなし・画像だけのウィンドウが自動で浮かび上がり、ドラッグで自由に動かせます
- **2種類の透明度スライダー** — 「YouTube/写真/動画」と「UI(ボタン類)」を別々に透明にできます
- **賢いウィンドウの重なり** — FloatPlayerがアクティブな間だけ最前面に浮き、他のアプリを使っている間は自動的に背面へ回ります(クリックすればまた前面に)
- **チャプター抽出** — YouTube Data API v3を使い、動画概要欄のタイムスタンプからチャプター一覧を自動抽出。メニューバーから選ぶと再読み込みなしでジャンプできます
- **メニューバー常駐** — Dockアイコンからでも、メニューバーのアイコンからでも操作可能。パネルを閉じてもアプリは常駐し続けます

## ビルド・起動

```sh
git clone https://github.com/HaruBoo/FloatPlayer.git
cd FloatPlayer
./build_app.sh
open FloatPlayer.app
```

`build_app.sh`は`swift build`の成果物を`.appバンドル`として組み立て、アドホック署名まで行います(署名はローカル実行専用で、配布はできません)。

以降は`FloatPlayer.app`をDockにドラッグして固定するか、Finderから直接起動できます。

### リビルド時の注意

ローカルのアドホック署名はビルドごとにハッシュ値が変わるため、スクリーンショット貼り付けなどでmacOSの権限ダイアログ(デスクトップフォルダへのアクセス等)が再度表示されることがあります。出てきたら「許可」を選んでください。

## 使い方

### 基本操作

- 上部のセグメントで **YouTube / 写真 / 動画** を切り替え(3つのモードは裏側で同時にマウントされているため、切り替えても再生位置は保たれます)
- 下部の2本のスライダーで、それぞれ **映像(YouTube/写真/動画)** と **UI(ボタン類)** の透明度を個別に調整
- 「クリックスルー」にチェックすると、ウィンドウの下にあるアプリを直接クリックできるようになります
- YouTube再生を開始すると自動的にUIが隠れ、映像だけの表示になります。メニューバーの「UIを表示」でいつでも操作パネルに戻せます

### YouTube再生

1. 「YouTube」タブでURLまたは動画IDを入力し「再生」を押す
2. 再生中に他のモードへ切り替えても、裏側で再生され続けます(自動で一時停止・再開)
3. メニューバーの「チャプター」から、抽出されたタイムスタンプへジャンプできます(下記のAPIキー設定が必要です)

### チャプター機能を使う場合

チャプター抽出にはYouTube Data API v3のAPIキーが必要です(無料枠で十分です)。

1. [Google Cloud Console](https://console.cloud.google.com/)でプロジェクトを作成
2. 「YouTube Data API v3」を有効化
3. 「認証情報」→「APIキーを作成」。アクセス範囲は **「一般公開データ」** を選択(ユーザーデータへのアクセスは不要です)
4. アプリのYouTube入力欄の下にあるAPIキー欄に貼り付け(ローカルに保存され、次回以降入力不要になります)

概要欄に`0:00 イントロ`のようなタイムスタンプが2件以上ある動画で、数秒後に自動でチャプターが読み込まれます。

### 写真・スクリーンショット

- 「写真を選択」でファイルを選ぶか、ウィンドウにドラッグ&ドロップ
- 「貼り付け」ボタン、Cmd+Shift+V、またはメニューバーからクリップボードの画像を直接貼り付け可能

### 動画ファイル

- 「動画を選択」で保存済みの動画ファイルを選ぶか、ウィンドウにドラッグ&ドロップ(自動ループ再生)

### スクリーンショットの自動フローティング

FloatPlayerを起動している間、Cmd+Shift+4などでスクリーンショットを撮ると、数百ミリ秒後に**UIなし・画像だけ**のウィンドウが自動で画面に浮かび上がります。

- ウィンドウの上をドラッグすると自由に移動できます
- 右クリックで「閉じる」「名前を付けて保存」「コピー」ができます
- メニューバーの「スクショを自動でフローティング表示」からON/OFFを切り替え可能

初回はデスクトップフォルダへのアクセス許可ダイアログが表示されます。「許可」を選んでください(スクリーンショットの保存先を変更している場合は、その設定を自動的に読み取ります)。

### メニューバー

メニューバーのアイコンから、以下を操作できます。

| 項目 | 内容 |
|---|---|
| パネルを表示 | ウィンドウを最前面に呼び戻す |
| クリックスルー | ON/OFF切り替え |
| UIを表示/隠す | ボタン類の表示切り替え |
| チャプター | 抽出済みチャプターへジャンプ |
| スクリーンショットを貼り付け | クリップボードの画像を貼り付け |
| スクショを自動でフローティング表示 | ON/OFF切り替え |
| 終了 | アプリを完全に終了 |

## 技術構成

Swift 6 / Swift Package Manager(Xcode不要) / AppKit(NSPanel・NSStatusItem) / SwiftUI(画面のUI) / WebKit(WKWebViewでYouTube再生) / AVFoundation(ローカル動画再生) / Combine(状態管理) / URLSession(YouTube Data API)

このアプリを作る過程で実際に使った技術や、実際に踏んだバグ(ウィンドウの巨大化、突然の終了など)については[学習ノート](docs/learning-notes/index.html)にまとめています。

## 既知の制限

- YouTube側で埋め込みが許可されていない動画は再生できません
- スクリーンショットの自動検出は、保存先フォルダの変化を監視する方式のため、クラウドストレージ等に直接保存する設定にしていると動作しない場合があります
- アドホック署名のため、初回起動時にGatekeeperの警告が出ることがあります(ローカルビルドのため想定内の挙動です)
