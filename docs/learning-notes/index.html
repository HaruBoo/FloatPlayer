FloatPlayer 学習ノート

macOS用フローティングプレイヤーを作りながら学んだ、Swift・AppKit・SwiftUI・WebKitの実践知識まとめです。一般的な入門ではなく、実際にこのアプリで使い、そして実際に踏んだバグだけを扱います。

FloatPlayerとは

FloatPlayerは、macOSの画面上に常に最前面で浮かぶ、小さなフローティングプレイヤーです。 YouTube動画・写真(スクリーンショット等)・保存済みの動画ファイルを、他のアプリで作業しながら 同時に見続けられるようにするためのユーティリティアプリです。

  • YouTube再生 — 公式のIFrame Player APIを使って埋め込み再生(ダウンロードはしません)。メニューバーからチャプター一覧にジャンプできます
  • 写真/スクリーンショット表示 — ファイル選択・ドラッグ&ドロップ・クリップボード貼り付けに対応
  • 動画ファイルのループ再生 — 保存済みの動画をBGV(バックグラウンドビデオ)のように流せます
  • 透明度の調整 — 「映像/写真/動画」と「UI(ボタン類)」を別々のスライダーで透明にできます
  • スクリーンショットの自動フローティング — Cmd+Shift+4などで撮った直後、UIなしの画像だけのウィンドウが自動で浮かび上がり、ドラッグで動かせます
  • 常駐・省スペース設計 — メニューバーのアイコンから表示/非表示・クリックスルー・終了などを操作。自分の作業を邪魔しないよう、他のアプリを使っている間は自動的に背面へ回ります

Xcodeを使わず、Swiftのコマンドラインツールとbuild_app.shだけでビルドできる、 ネイティブのSwift/AppKit/SwiftUIアプリです。以下の学習ノートでは、このアプリを作る過程で 実際に使った技術と、実際に踏んだバグをまとめています。

この学習ノートについて

FloatPlayerはXcodeを使わず、Swift Package Managerだけで作ったmacOSアプリ。 「YouTube/写真/動画を常に最前面のウィンドウで再生する」という単純な目的のアプリだが、 実装の過程でウィンドウが画面いっぱいまで巨大化する、入力しようとするとアプリごと消える、 YouTube埋め込みがエラーになるなど、AppKit特有のかなり手強いバグに何度も遭遇した。 この学習ノートの各章は、実際にこのアプリで使った/作った機能の解説と、その機能を作る過程で 実際に踏んだバグとその直し方をセットで記録している。

Tips: このアプリの技術構成

Swift 6 / Swift Package Manager(Xcode不要) / AppKit(NSPanel・NSStatusItem) / SwiftUI(画面のUI) / WebKit(WKWebViewでYouTube再生) / AVFoundation(ローカル動画再生) / Combine(状態管理) / URLSession(YouTube Data API) という組み合わせ。GUIアプリとしては小さいが、 「常に最前面に浮くウィンドウ」を作ろうとすると必然的にAppKitの深い部分に触れることになる。
